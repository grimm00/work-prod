### When to Add Documents

- **After each phase:** Create phase learnings and dev-infra improvements
- **After major milestones:** Create reflection reports
- **After important conversations:** Create chatlog entries
- **When patterns emerge:** Document in opportunities

### Organization Principles

- **Hub-and-spoke:** README files serve as navigation hubs
- **Chronological:** Date-based naming for reflections and chatlogs
- **Categorical:** Organize by type (opportunities, reflections, etc.)
- **Progressive disclosure:** Overview in hub, details in spoke documents
